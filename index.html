GitHub + Hexo 搭建属于自己的个人博客

发表于 2017-04-07

唠个五块钱的

使用 Hexo 之前我一直在用 WordPress，但是文章多了插件多了之后，最近实在是不堪忍受它的臃肿，所以就在物色着想要换个博客生成工具。
其实看中 Hexo 也挺久了，很中意它的轻量化和能免费部署到 GitHub 上，所以也一直想搭建一个基于 Hexo 的个人博客，但是因为我懒加上平时事情又多，所以就一直拖着。
昨晚无聊看了下 Hexo 的官方文档之后，想着既然看都看了，何不动手试试看！
然后虽然过程有点小曲折，但也没用多久就搭建起来了～
想着干脆记录下免得以后忘了细节再次踩坑，也顺便给想要用 GitHub + Hexo 搭建属于自己的个人博客的小伙伴们一个参考～

PS: 因为我使用的是Mac，所以操作内容都是以 MacOS 为参考，其实 Mac 与 Windows 搭建 GitHub 博客相差不大，用 Windows 的小伙伴可以跳过安装 Node 和 Git 的部分，Google 下这两个在 Windows 下的安装方法。

GitHub

  • 首先，我们得要有个 GitHub 的账号

    注册账号部分没什么好说的，直接进入 GitHub 官网注册。
    这个是注册地址：GitHub

  • 创建 GitHub 仓库

    登录账号后，在 Github 页面的右上方选择 New repository 进行仓库的创建。


    在仓库名字输入框中输入：

然后点击 Create repository 即可。


仓库创建成功之后，点击 Settings，下拉到 GitHub Pages
Source 选项选择 marster branch，点击 Save 保存
不出意外的话，你的 GitHub Pages 已经被发布到 https://yourname.github.io 并且可以访问了～

安装 Node.js

npm 在刚才安装 Node 的时候也已经一并安装成功，Git 的话 MacOS 已经自带。

配置 Git 个人信息


点击 New SSH Key 按钮，在 Key 处粘贴刚才复制的密钥，点击 Add SSH Key 提交密钥。
